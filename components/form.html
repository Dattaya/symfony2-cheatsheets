Symfony Form Component

Common options

Option Since Type Default Description Example
validation_groups 2.0 string, 2.1: callback It is used to specify which validation groups should be used to validate a form object.
mapped 2.1 bool true It is used in cases where you need extra fields in the form (for example: a "do you agree with these terms" checkbox) that will not be mapped to the underlying object. To achieve this set it to false.
property_path 2.0 string, false If you want to override the property/method name that a field reads from and writes to, you can set the property_path option. Its default value is the field's name.
data 2.0 mixed Field value of the underlying object Allows to override the initial value for the form or just an individual field.
disabled 2.0 bool false If you don't want a user to modify the value of a field, you can set the disabled option to true. Any submitted value will be ignored.
required 2.0 bool true If true, a HTML5 required attribute will be rendered.
trim 2.0 bool true If true, the whitespace of the submitted string value will be stripped via the trim() function when the data is bound.
attr 2.0 array Empty array If you want to add extra attributes to HTML field representation you can use attr option.
translation_domain 2.0 string messages This is the translation domain that will be used for any labels or options that are rendered for this field.
data_class 2.0 string Allows to set the name of the class that holds the underlying data. Required since 2.1 and if not set, an array is expected.
read_only 2.0 bool false In 2.0 if this option is true, the field will be rendered with the disabled attribute, since 2.1 with readonly="readonly"

Types

Text Fields

Type Since Description Options Example
text 2.0 The text field represents the most basic input text field.
textarea 2.0 Renders a textarea HTML element.
email 2.0 The email field is a text field that is rendered using the HTML5 <input type="email" /> tag.
integer 2.0 The input number field looks like a text box, except that - if the user's browser supports HTML5 - it will have some extra frontend functionality. rounding_mode, grouping
money 2.0 This field type allows you to specify a currency, whose symbol is rendered next to the text field. currency, divisor, precision, grouping
number 2.0 Renders an input text field and specializes in handling number input. precision, rounding_mode, grouping
password 2.0 The password field renders an input password text box. always_empty
percents 2.0 This field adds a percentage sign "%" after the input box. type, precision
search 2.0 This renders an <input type="search" /> field.
url 2.0 The url field is a text field that prepends the submitted value with a given protocol (e.g. http://) if the submitted value doesn't already have a protocol. default_protocol

Choice Fields

Type Since Description Options Example
choice 2.0 A multi-purpose field used to allow the user to "choose" one or more options. It can be rendered as a select tag, radio buttons, or checkboxes. choices, choice_list, multiple, expanded, preferred_choices, empty_value, empty_data
entity 2.0 Ships with the Doctrine Bridge. A special choice field that's designed to load options from a Doctrine entity. class, property, query_builder, em
country 2.0 The country type is a subset of the ChoiceType that displays countries of the world. The country names are displayed in the language of the user. The "value" for each country is the two-letter country code. The locale of your user is guessed using Locale::getDefault().
language 2.0 The language type is a subset of the ChoiceType that allows to select from a large list of languages. The language names are displayed in the language of the user. The "value" for each language is the Unicode language identifier (e.g. fr or zh-Hant). The locale of your user is guessed using Locale::getDefault().
locale 2.0 The locale type is a subset of the ChoiceType that allows to select from a large list of locales (language+country). The locale names are displayed in the language of the user. The "value": two letter ISO639-1 language code (e.g. fr), + optional underscore (_), and then the ISO3166 country code (e.g. fr_FR).
timezone 2.0 The timezone type is a subset of the ChoiceType that allows the user to select from all possible timezones. The "value" for each timezone is the full timezone name, such as America/Chicago.

Date and Time Fields

Type Since Description Options Example
date 2.0 A field that allows the user to modify date information via a variety of different HTML elements. widget, input, empty_value, years, months, days, format, pattern, data_timezone, user_timezone
datetime 2.0 This field type allows the user to modify data that represents a specific date and time (e.g. 1984-06-05 12:15:30). widget, time_widget, input, date_format, hours, minutes, seconds, years, months, days, with_seconds, data_timezone, user_timezone
time 2.0 A field to capture time input. widget, input, with_seconds, hours, minutes, seconds, data_timezone, user_timezone
birthday 2.0 This type is essentially the same as the date type, but with a more appropriate default for the years option. The years option defaults to 120 years ago to the current year. years

Other Fields

Type Since Description Options Example
checkbox 2.0 Creates a single input checkbox. value
file 2.0 The file type represents a file input in your form.
radio 2.0 Creates a single radio button. The radio type isn't usually used directly. More commonly it's used internally by other types such as choice. value

Groups

Type Since Description Options Example
collection 2.0 This type is used to render a "collection" of some form. In the easiest sense, it could be an array of text fields that populate an array emails field. In more complex examples, you can embed entire forms, which is useful when creating forms that expose one-to-many relationships (e.g. a product from where you can manage many related product photos). type, options, allow_add, allow_delete, prototype
repeated 2.0 This is a special field "group", that creates two identical fields whose values must match (or a validation error is thrown). The most common use is when you need the user to repeat his or her password or email to verify accuracy. type, options, first_name, second_name, 2.1: first_options, second_options

Hidden Fields

Type Since Description Options Example
hidden 2.0 The hidden type represents a hidden input field.
csrf 2.0 The csrf type is a hidden input field containing a CSRF token. csrf_provider, intention
