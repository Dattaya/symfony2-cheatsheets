Basic Constraints

Constraint Since Description Options Example
NotBlank 2.0 Validates that a value is not blank, defined as not equal to a blank string and also not equal to null. message
Blank 2.0 Validates that a value is blank, defined as equal to a blank string or equal to null. message
NotNull 2.0 Validates that a value is not strictly equal to null. message
Null 2.0 Validates that a value is exactly equal to null. message
True 2.0 Validates that a value is true. message
False 2.0 Validates that a value is false. message
Type 2.0 Validates that a value is of a specific data type. Supported types: array, bool, callable, float, double, int, integer, long, null, numeric, object, real, resource, scalar, string. message

Date Constraints

Constraint Since Description Options Example
Date 2.0 Validates that a value is a valid date, meaning either a DateTime object or a string (or an object that can be cast into a string) that follows a valid YYYY-MM-DD format. message
DateTime 2.0 Validates that a value is a valid "datetime", meaning either a DateTime object or a string (or an object that can be cast into a string) that follows a valid YYYY-MM-DD HH:MM:SS format. message
Time 2.0 Validates that a value is a valid time, meaning either a DateTime object or a string (or an object that can be cast into a string) that follows a valid HH:MM:SS format. message

Collection Constraints

Constraint Since Description Options Example
Choice 2.0 This constraint is used to ensure that the given value is one of a given set of valid choices. It can also be used to validate that each item in an array of items is one of those valid choices. choices, callback, multiple
Collection 2.0 This constraint is used when the underlying data is a collection (i.e. an array or an object that implements Traversable and ArrayAccess), but you'd like to validate different keys of that collection in different ways. fields, allowExtraFields
UniqueEntity 2.0 Validates that a particular field (or fields) in a Doctrine entity is (are) unique. This is commonly used, for example, to prevent a new user to register using an email address that already exists in the system. fields, message, em
Language 2.0 Validates that a value is a valid language code. message
Locale 2.0 Validates that a value is a valid locale. The "value" for each locale is either the two letter ISO639-1 language code (e.g. fr), or the language code followed by an underscore (_), then the ISO3166 country code (e.g. fr_FR for French/France). message
Country 2.0 Validates that a value is a valid two-letter country code. message
Size 2.1 Validates that a given string length or collection elements count is between some minimum and maximum value. It is possible to define only min or only max or both of them. min, max, type, charset, minMessage, maxMessage, exactMessage

File Constraints

Constraint Since Description Options Example
File 2.0 Validates that a value is a valid "file", which can be one of the following: A string (or object with a __toString() method) path to an existing file; A valid File object (including objects of class UploadedFile). maxSize, mimeTypes, maxSizeMessage, mimeTypesMessage, notFoundMessage, notReadableMessage, uploadIniSizeErrorMessage, uploadFormSizeErrorMessage, uploadErrorMessage
Image 2.0 The Image constraint works exactly like the File constraint, except that its mimeTypes and mimeTypesMessage options are automatically setup to work for image files specifically. mimeTypes, mimeTypesMessage, 2.1: minWidth, maxWidth, minHeight, maxHeight, sizeNotDetectedMessage, maxWidthMessage, minWidthMessage, maxHeightMessage, minHeightMessage

Other Constraints

Constraint Since Description Options Example
Callback 2.0 The purpose of the Callback assertion is to let you create completely custom validation rules and to assign any validation errors to specific fields on your object. A callback method itself doesn't fail or return any value. Instead a callback method has the ability to directly add validator "violations". methods
All 2.0 When applied to an array (or Traversable object), this constraint allows you to apply a collection of constraints to each element of the array. constraints
UserPassword 2.1 This validates that an input value is equal to the current authenticated user's password. This is useful in a form where a user can change his password, but needs to enter his old password for security. message
Valid 2.0 This constraint is used to enable validation on objects that are embedded as properties on an object being validated. This allows you to validate an object and all sub-objects associated with it. traverse
